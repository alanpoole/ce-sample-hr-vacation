Student Laboratory Handouts & Challenges

Welcome to the Time-Off Subsystem Lab! In this scenario, you are tasked with upgrading this baseline application from its single-region isolation into a globally available, fault-tolerant infrastructure pattern. Use these challenges to practice drafting production-ready cloud components.

GCP ARCHITECT Duration: 20 mins

Challenge 1: Design Multi-Region Relational Failover

The current database (Cloud SQL) runs as a standalone, single-region instance in us-central1 with no replication. If the entire US-Central datacenter encounters a catastrophic outage, the HR portal is dead. Upgrade your architecture by configuring high availability (HA) and cross-region replica configurations in Terraform. 

resource "google_sql_database_instance" "replica" {
  name                 = "hr-sql-replica-europe"
  region               = "europe-west1"
  database_version     = "POSTGRES_15"
  master_instance_name = google_sql_database_instance.primary.name

  settings {
    tier = "db-f1-micro"
    backup_configuration {
      binary_log_enabled = true
    }
  }
}
NETWORK DEV Duration: 30 mins

Challenge 2: Multi-Region Load Balancing & Anycast Routing

Currently, the frontend and backend are hardcoded to a single Cloud Run region. Submitters from Germany or Japan experience high network latency when routing traffic across the Atlantic. Integrate a **Global HTTPS Load Balancer** with Serverless NEGs (Network Endpoint Groups) to automatically distribute and resolve routes globally. 

# Sample Serverless NEG mapping
resource "google_compute_region_network_endpoint_group" "neg_us" {
  name                  = "neg-us-frontend"
  network_endpoint_type = "SERVERLESS"
  region                = "us-central1"
  cloud_run {
    service = "hr-frontend"
  }
}
DEVOPS ENGINEER Duration: 45 mins

Challenge 3: Active-Active Workflow Synced Firestore

Firestore has multi-region options that run on an active-active model (replicating records inside US or Europe dual-regions). Explore changing the Firestore database resource from nam5 (US Multi-region) to eur3 (Europe Multi-region) inside your Terraform module and document how replication lag impacts live locking rules.
